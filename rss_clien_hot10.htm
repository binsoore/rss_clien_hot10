설치하고 놀리고 계신 분은 한번 테스트 해보세요.</p> <p><br/></p> <h2>Ollama + MCP AI Agent 설치</h2> <p>일단 ollama 가 설치된 상태라고 가정하겠습니다. 설치가 안되어 있다면 아래 링크에서 다운로드 받아서 설치</p> <p>- <a href="https://ollama.com/" ori-url="https://ollama.com/" rel="noopener noreferrer" target="_blank">https://ollama.com/</a></p> <p><br/></p> <p>예제코드는 python 으로 작성되어 있고 uv 를 이용한 가상환경 설치를 지원합니다.</p> <p><strong>소스코드 target="_blank">https://github.com/godstale/ollama-mcp-agent</a> cd ollama-mcp-agent</pre> <p><br/></p> <p>예제코드 실행에 필요한 라이브러리 설치가 필요합니다. ollama-mcp-agent 예제는 uv 패키지 관리자를 지원합니다. termial 또는 PowerShell (terminal 환경에서 실행)</p> <pre># Create and activate virtual environment uv venv # For Unix-like systems source .venv/bin/activate # Or for Windows .venv\Scripts\activate # Install dependencies uv pip install -r requirements.txt</pre> <p><br/> ollama 에 LLM 모델을 다운로드해야 합니다. 이때 주의할 점이 하나가 있습니다. 반드시 tools calling 기능을 지원하는 모델을 받아야 합니다. 그렇지 않으면 MCP 설정을 추가해도 LLM 이 도구를 사용하지 않습니다.</p> <ul> <li>2025.04 기준으로 ollama 에서는 MFDoom/deepseek-r1-tool-calling 모델이 적합해 보입니다. <ul> <li>다른 tool 기능 활성화 된 모델은 제대로 동작하지 않았습니다.</li> </ul></li> <li>그래픽카드 메모리 용량에 맞춰 모델을 선택세요. (ollama 모델 페이지 참고) <ul> <li><a href="https://ollama.com/MFDoom/deepseek-r1-tool-calling" ori-url="https://ollama.com/MFDoom/deepseek-r1-tool-calling" rel="noopener" target="_blank" title="">https://ollama.com/MFDoom/deepseek-r1-tool-calling</a></li> </ul></li> <li>이 포스트에서는 RTX4070 SUPER 12G 그래픽카드에 맞춰 14b (9.0GB) 모델을 사용합니다.</li> </ul> <pre># Install Ollama (refer to <a href="https://ollama.ai" target="_blank">https://ollama.ai</a> for platform-specific installation) # Download LLM model which supports Tool calling feature ollama pull MFDoom/deepseek-r1-tool-calling:14b</pre> <p><br/></p> <h2>Ollama + MCP AI Agent 실행</h2> <p>아래 명령어로 LLM 을 실행하고 "오늘 서울의 날씨"에 대해 물어보면 당연하게도 답변을 못합니다.</p> <pre>ollama run MFDoom/deepseek-r1-tool-calling:14b </pre> <p><br/></p> <p>그럼 MCP 날씨 도구를 추가해서 ollama 와 연동하면 답변을 할 수 있는지 확인해 보겠습니다.</p> <p>소스코드에는 MCP 날씨 도구가 PC에서 실행될 수 있도록 python 코드로 미리 작성되어 있습니다. mcp_server/mcp_server_weather.py 파일을 보시면 됩니다. </p> <p>실제 날씨를 검색하지는 않고, 정해진 답만 리턴하도록 되어 있습니다. LLM 이 정해진 답을 사용하는지를 보면 됩니다.</p> <pre>from mcp.server.fastmcp import FastMCP mcp = FastMCP(     "Weather",  # Name of the MCP server     instructions="You are a weather assistant that can answer questions about the weather in a given location.",     host="0.0.0.0",  # Host address (0.0.0.0 allows connections from any IP)     port=8005,  # Port number for the server ) @mcp.tool() async def get_weather(location: str) -&amp;gt; str:     print(f"\n[DEBUG] MCP: get_weather called: {location}\n")     return f"It's sunny in the morning, cloudy in the afternoon and snowing in the evening in {location}" if __name__ == "__main__":     mcp.run(transport="stdio")</pre> <p>@mcp.tool() 어노테이션이 달린 함수를 LLM 이 도구로 인식합니다.
